# 开发规范

## DDD 四层架构

每个模块按 4 层组织，四层代码放在 `internal/` 子目录中，对外接口放在 `svc/` 包，RPC 适配放在顶层 `rpc/`。以 `module/player/` 为规范示例：

```
module/player/
├── internal/
│   ├── domain/                # 领域层（核心，不依赖任何外层）
│   │   ├── aggregate.go       # 聚合根 + 行为方法
│   │   ├── value_objects.go   # 值对象定义 + 校验
│   │   ├── events.go          # 领域事件定义
│   │   ├── repository.go      # 仓储接口
│   │   └── service.go         # 领域服务（纯函数）
│   ├── application/           # 应用层（编排，不包含领域规则）
│   │   ├── commands.go        # 命令定义
│   │   ├── handlers.go        # CommandHandler 实现
│   │   └── dto.go             # proto ↔ 领域对象转换
│   ├── infrastructure/        # 基础设施层
│   │   └── repo_memory.go     # 内存仓储实现
│   ├── interface/             # 接口层（最薄层）
│   │   └── routes.go          # 路由处理器
├── svc/                       # 对外接口和跨模块 Service
│   ├── interface.go           # DTO + interface
│   └── server/                # 跨模块 Service 实现
│       └── server.go          # 实现 svc.IPlayer 接口
├── rpc/                       # RPC 适配（如有）
│   ├── player.proto           # RPC proto 定义（如有）
│   ├── client/                # RPC 客户端
│   │   └── client.go
│   ├── server/                # RPC 服务端
│   │   └── server.go
│   └── grpc/                  # proto 生成代码
│       ├── player.pb.go
│       └── player_grpc.pb.go
├── module.go                  # Module 构造函数 + 依赖注入装配
├── option.go                  # 函数式选项
```

**设计理由**：`internal/` 是 Go 编译器的强制边界——外部包无法导入 `internal/` 下的任何代码，确保四层实现不被跨模块直接引用。只有 `svc/` 接口、`svc/server` 实现和 `rpc/` 适配对外暴露。

### 依赖方向

```
Interface ──→ Application ──→ Domain ←── Infrastructure
```

- **Domain** 不依赖任何外层
- **Application** 依赖 Domain，不依赖 Infrastructure（通过接口注入）
- **Interface** 解析 proto，构建命令，调用 Application
- **Infrastructure** 实现 Domain 定义的仓储接口

## `stack.Module` 接口

```go
type Module interface {
    Name() string
    Init(proxy *node.Proxy) error
}
```

`Init()` 按以下顺序装配：

1. 解析选项（`defaultOptions()` → `opt(o)`）
2. 创建领域层组件（`ddd.NewEventBus()`、`ddd.NewCommandBus()`）
3. 注册命令处理器到 CommandBus
4. 创建接口层路由处理器
5. 通过 `proxy.AddRouteHandler()` 注册路由
6. 通过 `actor.RegisterRouteInitializer()` 注册 Actor 路由（如有）
7. 注册 RPC 服务（如有）
8. 通过 `stack.RegisterService()` 暴露跨模块服务
9. 注册到 debug 服务（`debug.Register()`）

## module.go 模板

```go
package example

import (
	"github.com/dobyte/due/v2/cluster/node"
	"github.com/dobyte/due/v2/log"

	"github.com/skeletongo/game-stack/ddd"
	"github.com/skeletongo/game-stack/module/actor"
	"github.com/skeletongo/game-stack/module/example/internal/application"
	"github.com/skeletongo/game-stack/module/example/internal/domain"
	interfaces "github.com/skeletongo/game-stack/module/example/internal/interface"
	rpcserver "github.com/skeletongo/game-stack/module/example/rpc/server"
	svcserver "github.com/skeletongo/game-stack/module/example/svc/server"
	"github.com/skeletongo/game-stack/stack"
	"github.com/skeletongo/game-stack/stack/debug"
)

const name = "example"

func Module(opts ...Option) stack.Module {
    return &exampleModule{opts: opts}
}

type exampleModule struct {
    opts []Option
}

func (m *exampleModule) Name() string { return name }

func (m *exampleModule) Init(proxy *node.Proxy) error {
    o := defaultOptions()
    for _, opt := range m.opts {
        opt(o)
    }

    // ---- 基础设施层 ----
    repo := o.repo

    // ---- 领域层 ----
    eventBus := ddd.NewEventBus()
    cmdBus := ddd.NewCommandBus()

    // ---- 应用层：注册命令处理器 ----
    ddd.Register(cmdBus, application.CmdDoSomething, &application.DoSomethingHandler{
        Repo: repo, EventBus: eventBus,
    })

    // ---- 接口层 ----
    routes := interfaces.NewRouteHandlers(cmdBus, repo)

    // 查询路由（不走 Actor）
    proxy.AddRouteHandler(stack.RouteExampleQuery, routes.HandleQuery, stack.StatefulAuthorizedRoute)

    // 写操作路由（走 Actor 串行化）
    proxy.AddRouteHandler(stack.RouteExampleWrite,
        actor.RouteToActor(actor.KindPlayer), stack.StatefulAuthorizedRoute)

    actor.RegisterRouteInitializer(func(act *node.Actor) {
        act.AddRouteHandler(stack.RouteExampleWrite, routes.HandleWriteActor)
    })

    // ---- RPC 服务（供其他节点调用）----
    rpcserver.Register(name, proxy, repo)

    // ---- 跨模块 Service ----
    stack.RegisterService(name, svcserver.New(repo, cmdBus, proxy))

    // ---- Debug 服务（运行时查询/修改数据）----
    debug.Register[*domain.ExampleAggregate](name, repo, cmdBus)

    log.Infof("[%s] module initialized (DDD)", name)
    return nil
}

```

## 路由处理器模式

### 查询路由（不走 Actor）

纯查询通过 CommandBus 分发，不需要 Actor 串行化：

```go
func (h *RouteHandlers) HandleGetInfo(ctx node.Context) {
    req := &protocol.GetInfoRequest{}
    if err := ctx.Parse(req); err != nil {
        log.Errorf("parse failed: %v", err)
        stack.ProtoResponse(ctx, &protocol.GetInfoResponse{Code: stack.ErrCode(err), Message: err.Error()})
        return
    }
    pid := req.PlayerId
    if pid == 0 {
        pid = ctx.UID()
    }
    p, err := ddd.Dispatch[*domain.Player](ctx.Context(), h.CmdBus,
        application.GetPlayerCmd{PlayerID: ctx.UID(), TargetID: pid})
    if err != nil {
        log.Errorf("load failed: %v", err)
        stack.ProtoResponse(ctx, &protocol.GetInfoResponse{Code: stack.ErrCode(err), Message: err.Error()})
        return
    }
    stack.ProtoResponse(ctx, &protocol.GetInfoResponse{Code: stack.CodeOK, Player: application.PlayerToProto(p)})
}
```

### 写路由（走 Actor 串行化）

两步注册（与之前相同）：

```go
// ① Init 中注册 Node 路由（投递到 Actor mailbox）
proxy.AddRouteHandler(stack.RouteExampleWrite,
    actor.RouteToActor(actor.KindPlayer), stack.StatefulAuthorizedRoute)

// ② 注册 Actor 路由初始化器（Spawn 时自动应用）
actor.RegisterRouteInitializer(func(act *node.Actor) {
    act.AddRouteHandler(stack.RouteExampleWrite, routes.HandleWriteActor)
})
```

Actor handler 构建命令 → CommandBus 分派（Dispatch 返回 `(any, error)`）：

```go
func (h *RouteHandlers) HandleWriteActor(ctx node.Context) {
    req := &protocol.XxxRequest{}
    ctx.Parse(req)

    cmd := application.DoSomethingCmd{PlayerID: ctx.UID(), Param: req.Param}
    if _, err := h.CmdBus.Dispatch(ctx.Context(), cmd); err != nil {
        log.Errorf("dispatch failed: %v", err)
        stack.ProtoResponse(ctx, &protocol.XxxResponse{Code: stack.ErrCode(err), Message: err.Error()})
        return
    }
    stack.ProtoResponse(ctx, &protocol.XxxResponse{Code: stack.CodeOK})
}
```

有返回值的命令使用类型安全的 `ddd.Dispatch[T]`：

```go
result, err := ddd.Dispatch[*application.LoginResult](ctx.Context(), h.cmdBus, cmd)
```

无返回值的命令返回 `ddd.NoResult`，用 `_` 丢弃：

```go
_, err := h.cmdBus.Dispatch(ctx.Context(), cmd)
```

## 事件处理器（due 事件）

事件处理器不能调用 `ctx.Response()`。普通断线不会解绑玩家节点，也不会迁移 Actor；业务模块需要判断玩家是否在线时，优先使用 due 的定位接口（如 `LocateGate`）查询活跃连接，不自行维护在线状态字段。

```go
// 只有需要连接即时状态更新时，才通过 stack.AddEventHandler 注册，
// 并在 handler 内通过 LocateNode 校验当前节点归属权。
```

Disconnect 事件不应修改节点绑定。已有玩家节点归属只允许由显式玩家迁移流程修改。Repository 的缓存释放策略由仓储实现自己管理。

## 跨模块通信

### 同节点：Service 接口

模块通过 `svc/server/server.go` 实现 `svc/interface.go` 中定义的接口，注册到 `stack.RegisterService`：

```go
// player 模块注册
stack.RegisterService(name, svcserver.New(repo, cmdBus, proxy))

// 其他模块调用
svc := stack.GetService("player").(playersvc.IPlayer)
err := svc.AddExp(ctx, uid, 100)  // 在玩家 Actor 中同步执行，返回最新经验值
```

所有写操作通过 `InvokePlayerSync` 在目标玩家 Actor 中同步串行执行，调用方阻塞等待结果。

### 同节点：Actor 串行化

**异步 fire-and-forget**（不关心返回值）：

```go
actor.InvokePlayer(ctx, proxy, uid, func(ctx context.Context) {
    _ = playerSvc.AddExp(ctx, uid, 100)
})
```

**同步执行并获取结果**（推荐用于跨模块 Service）：

```go
newExp, err := actor.InvokePlayerSync[int64](ctx, proxy, uid, func(ctx context.Context) (int64, error) {
    return ddd.Dispatch[int64](ctx, cmdBus, application.AddExpCmd{...})
})
```

### 跨节点：EventBus + RPC

详见 `docs/用户数据并发修改安全设计.md` 和 `docs/due-api.md`。

## 响应模式

proto codec 下通过 `stack.ProtoResponse(ctx, msg)` 发送 proto 消息。

每个 Response 前两个字段固定为 `code`(int32) + `message`(string)。
成功时用 `stack.CodeOK`，错误时通过 `stack.ErrCode(err)` 从 `*stack.Code` 解包错误码。
领域层直接返回 `stack.Err*` 变量，接口层无需手动映射：

```go
// 成功
stack.ProtoResponse(ctx, &auth.LoginResponse{Code: stack.CodeOK, Token: token, PlayerId: uid})

// 错误：stack.ErrCode 自动提取 *Code 中的错误码
log.Errorf("xxx failed: %v", err)
stack.ProtoResponse(ctx, &auth.LoginResponse{Code: stack.ErrCode(err), Message: err.Error()})
```

`stack.Response` 封装仅适用于 JSON codec（`stack.JSONRespond*` 系列函数）。proto codec 下该结构体不实现 `proto.Message` 会导致序列化失败。

## 添加新模块流程

1. **战略设计** — 确定属于核心域/支撑域/通用域
2. **聚合设计** — 识别聚合根、实体、值对象，定义不变量
3. **创建目录结构** — `internal/domain/`、`internal/application/`、`internal/infrastructure/`、`internal/interface/`
4. **对外接口** — `svc/interface.go`（接口 + DTO）
5. **domain 层** — `value_objects.go` → `aggregate.go` → `events.go` → `repository.go` → `service.go`
6. **infrastructure 层** — `repo_memory.go`（至少提供内存实现）
7. **application 层** — `commands.go` → `handlers.go` → `dto.go`
8. **interface 层** — `routes.go`（路由处理器）
9. **svc/server** — 实现 `svc/` 中的接口，封装 Actor 串行化调用
10. **rpc/client + rpc/server** — RPC 客户端和服务端（如有）
11. **module.go** + `option.go` — 依赖注入装配
12. **proto 定义** — 客户端协议放在 `proto/`，模块 RPC 协议放在 `module/<name>/rpc/<name>.proto`
13. **注册** — 导入模块包并添加到 `cmd/node/main.go` 的 `stack.WithModules()`
