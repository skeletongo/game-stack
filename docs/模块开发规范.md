# 开发规范

## 模块开发

### 6 文件模式

每个模块包固定由 6 个文件组成，以 `module/auth/` 为规范示例：

| 文件 | 用途 | 关键要点 |
|------|------|----------|
| `store.go` | 数据类型 + `Store` 接口 | 抽象数据持久化，所有方法接受 `context.Context` |
| `store_memory.go` | 默认内存 `Store` 实现 | `map` + `sync.RWMutex`，仅用于开发环境 |
| `option.go` | 函数式选项 | `WithStore(s Store)` 注入自定义存储 |
| `service.go` | 跨模块 `Service` 接口 | 定义本模块对外暴露的能力 |
| `impl.go` | 路由/事件处理器实现 | `handleXxx(ctx node.Context)`，核心业务逻辑 |
| `module.go` | `Module()` 构造函数 | `Init()` 中注册路由、事件、服务 |

### `stack.Module` 接口

```go
type Module interface {
    Name() string
    Init(proxy *node.Proxy) error
}
```

`Init()` 在应用启动时被调用，按以下顺序操作：

1. 解析选项（`defaultOptions()` → `opt(o)` 应用用户配置）
2. 创建内部依赖（清理器、actor 等）
3. 通过 `proxy.AddRouteHandler()` 注册路由
4. 通过 `proxy.AddEventHandler()` 注册事件
5. 通过 `stack.RegisterService()` 暴露服务

### module.go 模板

```go
package example

import (
    "github.com/dobyte/due/v2/cluster"
    "github.com/dobyte/due/v2/cluster/node"
    "github.com/dobyte/due/v2/log"
    "github.com/skeletongo/game-stack/stack"
)

const name = "example"

func Module(opts ...Option) stack.Module {
    return &exampleModule{opts: opts}
}

type exampleModule struct {
    opts []Option
}

func (m *exampleModule) Name() string { return name }

func (m *exampleModule) Init(proxy *node.Proxy) error {
    o := defaultOptions()
    for _, opt := range m.opts {
        opt(o)
    }

    impl := newImpl(o.store, proxy)

    // 无需授权的路由
    proxy.AddRouteHandler(stack.RouteExampleXxx, impl.handleXxx)
    // 需要授权的路由
    proxy.AddRouteHandler(stack.RouteExampleYyy, impl.handleYyy, stack.StatefulAuthorizedRoute)

    // 连接/断线事件
    proxy.AddEventHandler(cluster.Connect, impl.handleConnect)
    proxy.AddEventHandler(cluster.Disconnect, impl.handleDisconnect)

    // 暴露服务给其他模块
    stack.RegisterService(name, impl.svc)

    log.Infof("[%s] module initialized", name)
    return nil
}
```

### impl.go 模式

每个路由处理器遵循统一结构。**写操作应通过 Actor 串行化**（`RouteToActor` 或 `actor.Invoke`），纯查询可走普通 handler：

```go
// 普通 handler（纯查询）
func (i *impl) handleXxx(ctx node.Context) {
    req := &protocol.XxxRequest{}
    if err := ctx.Parse(req); err != nil {
        stack.RespondError(ctx, stack.ErrInvalidParam)
        return
    }
    result, err := i.svc.DoSomething(context.Background(), req.Field)
    if err != nil {
        stack.RespondError(ctx, stack.ErrXxx)
        return
    }
    stack.RespondData(ctx, &protocol.XxxResponse{Result: result})
}
```

涉及状态修改的 handler 使用 Actor 模式，详见 `docs/用户数据并发修改安全设计.md`。

## 事件处理器

事件处理器**不能**调用 `ctx.Response()`。它用于监听连接生命周期，执行安全清理等副作用操作：

```go
func (i *impl) handleConnect(ctx node.Context) {
    log.Infof("player connected: uid=%d cid=%d", ctx.UID(), ctx.CID())
}

func (i *impl) handleDisconnect(ctx node.Context) {
    uid := ctx.UID()
    if uid != 0 {
        actor.KillPlayer(i.proxy, uid)      // 立即杀 Actor，关闭 mailbox
        i.cleaner.OnDisconnect(uid)          // 启动 Grace Period 延迟清理
    }
    log.Infof("player disconnected: uid=%d", uid)
}
```

## Store 模式

每个模块通过 `Store` 接口抽象数据持久化，默认使用内存实现，生产环境可替换为 Redis/MySQL/MongoDB等。详见 `docs/store设计.md`。

## Service 模式

模块通过 `Service` 接口暴露能力，在 `Init()` 中注册到全局服务注册表：

```go
// service.go —— 接口定义
type Service interface {
    DoSomething(ctx context.Context, param string) (*Result, error)
}
```

```go
// 在 Init() 中注册
stack.RegisterService(name, impl.svc)
```

```go
// 其他模块通过类型断言获取
svc, ok := stack.GetService("example").(example.Service)
if ok {
    result, err := svc.DoSomething(ctx, "param")
}
```

## 模块间通信

### 同节点：直接调用

通过 `stack.RegisterService` / `stack.GetService` + 类型断言。

### 同节点：Actor 串行化

对玩家状态的修改必须通过 Actor 串行化，避免多 goroutine 并发写导致状态不一致：

- **RouteToActor** — 需要同步响应的客户端请求
- **actor.Invoke** — 不需要同步响应的 fire-and-forget

```go
// 客户端请求（RouteToActor）
proxy.AddRouteHandler(route, actor.RouteToActor(actor.KindPlayer), stack.StatefulAuthorizedRoute)

// 模块间事件触发（Invoke）
actor.Invoke(proxy, uid, func() {
    playerSvc.AddGold(uid, amount)
})
```

详见 `docs/用户数据并发修改安全设计.md`。

### 跨节点：EventBus

通过 `stack/event.go` 中定义的 Topic 常量和 Redis EventBus 发布/订阅事件：

```go
// 发布
proxy.PublishEvent(stack.EventPlayerLevelUp, uid, newLevel)

// 订阅（在 Init 中）
proxy.AddEventHandler(stack.EventPlayerLevelUp, impl.handleLevelUp)
```

### 跨节点：RPC 调用

due 提供两种跨节点同步操作方式：

**消息投递** — 将消息投递到指定节点或用户所在节点，接收方通过路由处理器处理：

```go
// 投递到指定节点
proxy.Deliver(ctx, &cluster.DeliverArgs{
    NID:  targetNID,
    UID:  uid,
    Message: &cluster.Message{Route: routeID, Data: payload},
})
```

**gRPC 服务调用** — 注册 gRPC 服务并跨节点直连调用：

```go
// 服务端：在 Init 中注册
proxy.AddServiceProvider("player", &playerDesc{}, &playerService{})

// 调用方：通过服务发现或直连调用
client, _ := proxy.NewMeshClient("discovery://player")
reply, _ := client.Call(ctx, "GetPlayer", &GetPlayerReq{UID: uid})
```

详见 `docs/due-api.md`。

## 响应辅助函数

所有客户端响应必须使用 `stack` 包的辅助函数，确保统一信封格式：

| 函数 | 用途 |
|------|------|
| `stack.RespondOK(ctx)` | 成功，无数据 |
| `stack.RespondData(ctx, data)` | 成功，带数据 |
| `stack.RespondError(ctx, code)` | 错误，使用预定义错误码 |
| `stack.RespondErrorMsg(ctx, code, msg)` | 错误，带自定义消息 |

## 添加新模块步骤

1. 创建 `protocol/<name>/message.go` —— 请求/响应结构体（带 json/msgpack 标签）
2. 按照 6 文件模式创建 `module/<name>/`
3. 在 `stack/route.go` 中分配模块号，按 `模块号 × 1000 + 子协议号` 添加路由常量
4. 在 `stack/errcode.go` 中添加错误码
5. 在 `stack/event.go` 中添加需要的事件 Topic（如有）
6. 导入模块包并添加到 `cmd/node/main.go` 的 `stack.WithModules()` 中

## 入口点

### cmd/node/main.go

游戏节点入口，负责组装所有组件和模块：

```go
func main() {
    app := stack.NewApplication(
        stack.WithName("game-node"),
        stack.WithLocator(redis.NewLocator()),
        stack.WithRegistry(etcd.NewRegistry()),
        stack.WithTransporter(grpc.NewTransporter()),
        stack.WithModules(
            auth.Module(),
            player.Module(),
            // ... 其他模块
        ),
    )
    app.Run()
}
```

`app.Run()` 创建 due Container → 构建 Node → 初始化所有模块 → `container.Serve()` 阻塞直到 SIGTERM。

### cmd/gate/main.go

网关入口，负责 WebSocket 连接接入、消息路由转发。不加载游戏模块。
