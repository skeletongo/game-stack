# 开发规范

## DDD 四层架构

每个模块按 4 层组织，以 `module/player/` 为规范示例：

```
module/player/
├── domain/                    # 领域层（核心，不依赖任何外层）
│   ├── aggregate.go           # 聚合根 + 行为方法
│   ├── value_objects.go       # 值对象定义 + 校验
│   ├── events.go              # 领域事件定义
│   ├── repository.go          # 仓储接口
│   └── service.go             # 领域服务（纯函数）
├── application/               # 应用层（编排，不包含领域规则）
│   ├── commands.go            # 命令定义
│   ├── handlers.go            # CommandHandler 实现
│   └── dto.go                 # proto ↔ 领域对象转换
├── infrastructure/            # 基础设施层
│   └── repo_memory.go         # 内存仓储实现
├── interface/                 # 接口层（最薄层）
│   └── routes.go              # 路由处理器
├── module.go                  # Module 构造函数 + 依赖注入装配
├── option.go                  # 函数式选项
├── grpc/                      # gRPC 适配（如有）
│   ├── player.proto
│   ├── player.pb.go
│   ├── player_grpc.pb.go
│   └── server.go
```

### 依赖方向

```
Interface ──→ Application ──→ Domain ←── Infrastructure
```

- **Domain** 不依赖任何外层
- **Application** 依赖 Domain，不依赖 Infrastructure（通过接口注入）
- **Interface** 解析 proto，构建命令，调用 Application
- **Infrastructure** 实现 Domain 定义的仓储接口

## `stack.Module` 接口

```go
type Module interface {
    Name() string
    Init(proxy *node.Proxy) error
}
```

`Init()` 按以下顺序装配：

1. 解析选项（`defaultOptions()` → `opt(o)`）
2. 创建领域层组件（`ddd.NewEventBus()`、`ddd.NewCommandBus()`）
3. 注册命令处理器到 CommandBus
4. 创建接口层路由处理器
5. 通过 `proxy.AddRouteHandler()` 注册路由
6. 通过 `proxy.AddEventHandler()` 注册事件（如有）
7. 通过 `actor.RegisterRouteInitializer()` 注册 Actor 路由（如有）
8. 注册 gRPC 服务（如有）
9. 注册清理回调到 PlayerDoneCleaner
10. 通过 `stack.RegisterService()` 暴露跨模块服务

## module.go 模板

```go
package example

import (
    "github.com/dobyte/due/v2/cluster/node"
    "github.com/dobyte/due/v2/log"

    "github.com/skeletongo/game-stack/ddd"
    "github.com/skeletongo/game-stack/module/actor"
    "github.com/skeletongo/game-stack/module/example/application"
    "github.com/skeletongo/game-stack/module/example/domain"
    interfaces "github.com/skeletongo/game-stack/module/example/interface"
    "github.com/skeletongo/game-stack/stack"
)

const name = "example"

func Module(opts ...Option) stack.Module {
    return &exampleModule{opts: opts}
}

type exampleModule struct {
    opts []Option
}

func (m *exampleModule) Name() string { return name }

func (m *exampleModule) Init(proxy *node.Proxy) error {
    o := defaultOptions()
    for _, opt := range m.opts {
        opt(o)
    }

    // ---- 基础设施层 ----
    repo := o.repo

    // ---- 领域层 ----
    eventBus := ddd.NewEventBus()
    cmdBus := ddd.NewCommandBus()

    // ---- 应用层：注册命令处理器 ----
    ddd.Register(cmdBus, application.CmdDoSomething, &application.DoSomethingHandler{
        Repo: repo, EventBus: eventBus,
    })

    // ---- 接口层 ----
    routes := interfaces.NewRouteHandlers(cmdBus, repo)

    // 查询路由（不走 Actor）
    proxy.AddRouteHandler(stack.RouteExampleQuery, routes.HandleQuery, stack.StatefulAuthorizedRoute)

    // 写操作路由（走 Actor 串行化）
    proxy.AddRouteHandler(stack.RouteExampleWrite,
        actor.RouteToActor(actor.KindPlayer), stack.StatefulAuthorizedRoute)

    actor.RegisterRouteInitializer(func(act *node.Actor) {
        act.AddRouteHandler(stack.RouteExampleWrite, routes.HandleWriteActor)
    })

    // 清理回调
    if c, ok := stack.GetService("cleaner").(*stack.PlayerDoneCleaner); ok {
        c.Register(&cleanableAdapter{repo: repo})
    }

    // 跨模块 Service
    stack.RegisterService(name, &svcAdapter{repo: repo, cmdBus: cmdBus})

    log.Infof("[%s] module initialized (DDD)", name)
    return nil
}
```

## 路由处理器模式

### 查询路由（不走 Actor）

纯查询直接从仓储读取，不需要 Actor 串行化：

```go
func (h *RouteHandlers) HandleGetInfo(ctx node.Context) {
    req := &protocol.GetInfoRequest{}
    if err := ctx.Parse(req); err != nil {
        log.Errorf("parse failed: %v", err)
        ctx.Response(nil)
        return
    }
    p, err := h.GetInfo.GetPlayer(ctx.Context(), ctx.UID())
    if err != nil {
        log.Errorf("load failed: %v", err)
        ctx.Response(nil)
        return
    }
    ctx.Response(&protocol.GetInfoResponse{Player: application.PlayerToProto(p)})
}
```

### 写路由（走 Actor 串行化）

两步注册：

```go
// ① Init 中注册 Node 路由（投递到 Actor mailbox）
proxy.AddRouteHandler(stack.RouteExampleWrite,
    actor.RouteToActor(actor.KindPlayer), stack.StatefulAuthorizedRoute)

// ② 注册 Actor 路由初始化器（Spawn 时自动应用）
actor.RegisterRouteInitializer(func(act *node.Actor) {
    act.AddRouteHandler(stack.RouteExampleWrite, routes.HandleWriteActor)
})
```

Actor handler 构建命令 → CommandBus 分派：

```go
func (h *RouteHandlers) HandleWriteActor(ctx node.Context) {
    req := &protocol.XxxRequest{}
    ctx.Parse(req)

    cmd := application.DoSomethingCmd{PlayerID: ctx.UID(), Param: req.Param}
    if err := h.CmdBus.Dispatch(ctx.Context(), cmd); err != nil {
        log.Errorf("dispatch failed: %v", err)
        ctx.Response(nil)
        return
    }
    ctx.Response(nil)
}
```

## 事件处理器(due事件)

事件处理器不能调用 `ctx.Response()`。用于连接生命周期管理：

```go
func (h *Handlers) HandleConnect(ctx node.Context) {
    uid := ctx.UID()
    if uid == 0 { return }
    h.cleaner.OnLogin(uid)  // 取消延迟清理
    if act := actor.GetPlayer(h.proxy, uid); act == nil {
        actor.SpawnPlayer(h.proxy, uid)  // 恢复 Actor
    }
}

func (h *Handlers) HandleDisconnect(ctx node.Context) {
    uid := ctx.UID()
    if uid != 0 {
        actor.KillPlayer(h.proxy, uid)    // 立即杀 Actor
        h.cleaner.OnDisconnect(uid)        // 启动 Grace Period
    }
}
```

## 跨模块通信

### 同节点：Service 适配器

模块通过 `svcAdapter` 暴露能力，其他模块通过 `stack.GetService` 获取：

```go
// player 模块注册
stack.RegisterService("player", &svcAdapter{repo: repo, cmdBus: cmdBus})

// 其他模块调用
svc := stack.GetService("player").(*player.SvcAdapter)
err := svc.AddGold(uid, 100)
```

### 同节点：Actor 串行化

写操作必须通过 `actor.InvokePlayer` 投递到 Actor：

```go
actor.InvokePlayer(proxy, uid, func() {
    playerSvc.AddExp(uid, 100)
    questSvc.UpdateProgress(uid, questMatchWin, 1)
})
```

### 跨节点：EventBus + gRPC

详见 `docs/用户数据并发修改安全设计.md` 和 `docs/due-api.md`。

## 响应模式

proto codec 下直接用 `ctx.Response(protoMsg)` 发 proto 消息，不包信封。错误时发 `ctx.Response(nil)` 配合日志：

```go
// 成功带数据
ctx.Response(&auth.LoginResponse{Token: token, PlayerId: uid})

// 成功无数据
ctx.Response(nil)

// 错误
log.Errorf("xxx failed: %v", err)
ctx.Response(nil)
```

`stack.Response` 封装仅适用于 JSON codec。proto codec 下该结构体不实现 `proto.Message` 会导致序列化失败。

## 添加新模块流程

1. **战略设计** — 确定属于核心域/支撑域/通用域
2. **聚合设计** — 识别聚合根、实体、值对象，定义不变量
3. **创建目录结构** — `domain/`、`application/`、`infrastructure/`、`interface/`
4. **domain 层** — `value_objects.go` → `aggregate.go` → `events.go` → `repository.go` → `service.go`
5. **infrastructure 层** — `repo_memory.go`（至少提供内存实现）
6. **application 层** — `commands.go` → `handlers.go` → `dto.go`
7. **interface 层** — `routes.go`（路由处理器）
8. **module.go** + `option.go` — 依赖注入装配
9. **proto 定义** — `proto/<name>/<name>.proto` + `stack/route.go` + `stack/errcode.go`
10. **注册** — 导入模块包并添加到 `cmd/node/main.go` 的 `stack.WithModules()`
