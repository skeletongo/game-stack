# 客户端服务端通信协议设计

## 消息格式

支持两种 codec：

### Proto（当前默认）

每个 Response 消息前两个字段固定为 `code`(int32) + `message`(string)，默认 code=0,1 表示成功。

响应通过 `stack.ProtoResponse(ctx, msg)` 发送，错误码通过 `stack.ErrCode(err)` 从 `*stack.Code` 解包。

### JSON

统一信封格式：

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

响应通过 `stack.JSONRespond(ctx, resp)` 及辅助函数 `JSONRespondOK` / `JSONRespondData` / `JSONRespondError` 封装。

## 路由编号

采用 **`模块号 × 1000 + 子协议号`** 公式。`route / 1000` 得模块号，`route % 1000` 得子协议号。

路由常量统一定义在 `stack/route.go`。

### 模块号分配

| 模块号 | 模块 | 描述 | 区间 |
|--------|------|------|------|
| 0 | — | 系统内部路由（心跳、GM 指令等） | 0–999 |
| 1 | auth | 用户登录、鉴权、会话管理 | 1000–1999 |
| 2 | player | 玩家基础信息、属性管理 | 2000–2999 |

### 当前路由列表

#### Auth (1000–1999)

| 常量 | 值 | 说明 |
|------|-----|------|
| `RouteAuthLogin` | 1001 | 登录 |
| `RouteAuthRegister` | 1002 | 注册 |
| `RouteAuthLogout` | 1003 | 登出 |
| `RouteAuthTokenRefresh` | 1004 | 刷新令牌 |
| `RouteAuthKick` | 1005 | 踢人下线 |

#### Player (2000–2999)

| 常量 | 值 | 说明 |
|------|-----|------|
| `RoutePlayerGetInfo` | 2001 | 获取玩家信息 |
| `RoutePlayerSetAvatar` | 2005 | 设置头像 |
| `RoutePlayerLevelUp` | 2006 | 升级推送 |

## 错误码

错误码定义在 `stack/errcode.go`。使用 `&Code{Code: int32, Message: string}` 结构体，实现 `error` 接口。

### 系统错误 (0–999)

复用 HTTP 语义：

| 常量 | Code | 说明 |
|------|------|------|
| `ErrOK` | 0,1  | 成功 |
| `ErrInvalidParam` | 400  | 参数错误 |
| `ErrUnauthorized` | 401  | 未登录 |
| `ErrForbidden` | 403  | 无权限 |
| `ErrNotFound` | 404  | 资源不存在 |
| `ErrTimeout` | 408  | 请求超时 |
| `ErrInternalError` | 500  | 内部错误 |
| `ErrServerBusy` | 503  | 服务器繁忙 |
| `ErrDuplicateRequest` | 429  | 重复请求 |

### 业务错误 (1000+)

与路由编号使用相同的 `模块号 × 1000 + 子码` 公式，模块号分配完全一致。系统错误占用 0–999，每个模块各占用 1000 个号段。

| 模块号 | 模块 | 区间 |
|--------|------|------|
| 0 | 系统 | 0–999（HTTP 语义） |
| 1 | auth | 1000–1999 |
| 2 | player | 2000–2999 |
