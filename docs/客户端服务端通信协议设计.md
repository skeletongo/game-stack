# 客户端服务端通信协议设计

## 消息格式

所有请求/响应均采用 JSON 或 MessagePack 编码，统一信封格式：

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

响应通过 `stack.Respond(ctx, resp)` 及辅助函数 `RespondOK` / `RespondData` / `RespondError` 封装。

## 路由编号

采用 **`模块号 × 1000 + 子协议号`** 公式。`route / 1000` 得模块号，`route % 1000` 得子协议号。

路由常量统一定义在 `stack/route.go`。

### 模块号分配

| 模块号 | 模块 | 描述 | 区间 |
|--------|------|------|------|
| 0 | — | 系统内部路由（心跳、GM 指令等） | 0–999 |
| 1 | auth | 用户登录、鉴权、会话管理 | 1000–1999 |
| 2 | player | 玩家基础信息、属性管理 | 2000–2999 |
| 3 | chat | 消息发送、频道管理 | 3000–3999 |
| 4 | match | 对局匹配队列 | 4000–4999 |
| 5 | room | 房间创建、加入、离开 | 5000–5999 |
| 6 | inventory | 物品存储、使用、装备 | 6000–6999 |
| 7 | quest | 任务接取、进度推进、完成 | 7000–7999 |
| 8 | combat | 战斗逻辑与伤害计算 | 8000–8999 |
| 9 | guild | 公会创建、管理、成员维护 | 9000–9999 |
| 10 | mail | 邮件收发、附件领取 | 10000–10999 |
| 11 | shop | 商品浏览与购买 | 11000–11999 |
| 12 | leaderboard | 排名数据管理与查询 | 12000–12999 |
| 13 | activity | 限时活动参与与奖励 | 13000–13999 |
| 14 | social | 好友与关系链管理 | 14000–14999 |

## 错误码

错误码定义在 `stack/errcode.go`。使用 `&Code{Code: int32, Message: string}` 结构体，实现 `error` 接口。

### 系统错误 (0–999)

复用 HTTP 语义：

| 常量 | Code | 说明 |
|------|------|------|
| `ErrOK` | 0 | 成功 |
| `ErrInvalidParam` | 400 | 参数错误 |
| `ErrUnauthorized` | 401 | 未登录 |
| `ErrForbidden` | 403 | 无权限 |
| `ErrNotFound` | 404 | 资源不存在 |
| `ErrTimeout` | 408 | 请求超时 |
| `ErrInternalError` | 500 | 内部错误 |
| `ErrServerBusy` | 503 | 服务器繁忙 |
| `ErrDuplicateRequest` | 429 | 重复请求 |

### 业务错误 (1000+)

与路由编号使用相同的 `模块号 × 1000 + 子码` 公式，模块号分配完全一致。系统错误占用 0–999，模块 1–14 各占用 1000 个号段。
